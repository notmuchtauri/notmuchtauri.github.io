Architecture 🟦

Notmuch Tauri acts as the modern graphical interface (GUI) orchestrating a set of specialized, robust, and proven UNIX CLI tools. Here is how the components interact:

Notmuch Tauri agit comme l'interface moderne (GUI) qui orchestre un ensemble d'outils CLI UNIX spécialisés, robustes et éprouvés. Voici comment les composants interagissent :

           IMAP Server

            mbsync

    notmuch  afew   Tauri

        ┌─────────────┴─────────────┐
        │                           │
   LanguageTool                   msmtp

        ▼                          ▼ 
       LLM                          SMTP Server
  • mbsync: Downloads your emails from the IMAP server to a local Maildir format.
  • notmuch: Instantly indexes your Maildir for ultra-fast searching.
  • afew: Automatically applies tags (inbox, spam, lists) to your new emails.
  • msmtp: Reliably sends your emails via SMTP.
  • Tauri: The lightweight and fast graphical interface you use daily.
  • LanguageTool / LLM: Your AI assistants for proofreading and writing.
  • mbsync : Télécharge vos e-mails depuis le serveur IMAP vers un format Maildir local.
  • notmuch : Indexe instantanément votre Maildir pour une recherche ultra-rapide.

  • msmtp : Envoie vos e-mails (SMTP) de manière fiable.
  • Tauri : L'interface graphique légère et rapide que vous utilisez au quotidien.
  • LanguageTool / LLM : Vos assistants pour la correction et la rédaction (IA).

The core of the system. An incredibly fast and extensible tag-based email search engine.

Le cœur du système. Un moteur de recherche d'e-mails incroyablement rapide et extensible basé sur des tags.

afew Recommended Recommandé

A Python script that integrates with notmuch to automatically tag your emails upon receipt.

Un script Python qui s'intègre à notmuch pour taguer automatiquement vos e-mails lors de la réception.

Workflow 🔄

⚡ Quick Start: If you want to fetch your emails manually in the terminal:

⚡ Quick Start : Si vous voulez récupérer vos e-mails manuellement dans le terminal :

sem --fg -j 1 --id mbsync  '/usr/bin/mbsync -a' ; sem -j 1 --fg --id notmuch '/usr/bin/notmuch new --quiet'

⚡ Quick Start: If you want to put this script in your crontab:

FAQ ❓

Why use mbsync? Pourquoi utiliser mbsync ?

Unlike traditional email clients, mbsync fetches all your emails locally (Maildir). This allows instant indexing with notmuch and 100% offline availability.

Contrairement aux clients e-mails traditionnels, mbsync rapatrie tous vos e-mails localement (Maildir). Cela permet une indexation instantanée avec notmuch et une disponibilité 100% hors-ligne.

Why not direct IMAP? Pourquoi pas IMAP directement ?

IMAP is slow for searching. Notmuch can search through millions of emails in milliseconds because everything is indexed on your SSD.

IMAP est lent pour la recherche. Notmuch peut chercher à travers des millions d'e-mails en quelques millisecondes car tout est indexé sur votre disque SSD.

How to sync automatically? Comment synchroniser automatiquement ?

You can use a systemd timer, a cron job, or a tool like goimapnotify to trigger the Quick Start upon each new email received on the server.

Vous pouvez utiliser un timer systemd, un cron job, ou un outil comme goimapnotify pour déclencher le Quick Start à chaque nouvel e-mail reçu sur le serveur.

Can I use multiple accounts? Puis-je utiliser plusieurs comptes ?

Yes! Just add multiple Account sections in your .mbsyncrc and .msmtp/config files. Notmuch will group everything under a single unified search interface.

Oui ! Il suffit d'ajouter plusieurs sections Account dans vos fichiers .mbsyncrc et .msmtp/config. Notmuch regroupera le tout sous une seule interface de recherche unifiée.

How to use my own LLM? Comment utiliser mon propre LLM ?

Simply replace llm_endpoint in config.json with the URL of your local Ollama instance (e.g., http://localhost:11434/v1/chat/completions) and put a dummy API key.

Remplacez simplement llm_endpoint dans config.json par l'URL de votre instance Ollama locale (ex: http://localhost:11434/v1/chat/completions) et mettez une clé API factice.

Is LanguageTool mandatory? LanguageTool est-il obligatoire ?

No. It's an optional feature. If you leave the languagetool_url field empty, the interface will simply hide the grammar correction button.

Non. C'est une fonctionnalité optionnelle. Si vous laissez le champ languagetool_url vide, l'interface masquera simplement le bouton de correction grammaticale.
